## 📝 개발 규칙

### TypeScript
- ❌ any 타입 금지
- ✅ 모든 변수/함수 명시적 타입 정의
- ✅ strict mode: true

### 컴포넌트
- ✅ Server Component 기본값
- ⚠️ Client Component 필요시만 (`'use client'`)
- ✅ 단일 책임 원칙

### 코딩 스타일
- **들여쓰기**: 2칸 (스페이스)
- **네이밍**: camelCase (함수), PascalCase (컴포넌트)
- **주석**: 한글로 작성
- **포맷팅**: Prettier + ESLint 필수

### 파일 구조
```
src/
├── app/              # Next.js 페이지
├── components/       # React 컴포넌트
├── lib/              # 유틸리티 (utils, env, constants)
├── types/            # TypeScript 타입
├── services/         # API 통신 로직
└── hooks/            # 커스텀 훅
```
